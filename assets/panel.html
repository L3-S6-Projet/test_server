Mini documentation

Ce serveur persiste ses données dans un fichier db.json. On peut s'échanger ce fichier quand le serveur est éteint. Le lien de réinitialisation supprime toutes les données du serveur, puis ajoute les données de test.

L'authentification est gérée avec un système de token : on récupére un token à la connexion. On doit ensuite l'envoyer à chaque requête avec le header suivant:

Authorization: Bearer $TOKEN

Pour pouvoir utiliser les routes dans la documentation swagger, il faut cliquer sur "Authorize" en haut à droite, puis rentrer "Bearer $TOKEN" et cliquer sur "Authorize".

Par défaut, trois utilisateurs de test sont créé : user.admin, user.teacher et user.student.

Le mot de passe de chaque utilisteur créé par défaut est le même que son nom d'utilisateur.
